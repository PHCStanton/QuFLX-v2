## 2. CRITICAL Issues (Immediate Action Required)

### 2.1 🔴 CRITICAL – Silent Error Swallowing in Backend

**Location:** `backend/services/gateway/asset_control.py`  
**CORE_PRINCIPLES Violated:** #8 (Defensive & Explicit Error Handling)

**Finding:**  
Multiple bare `except:` blocks that swallow errors silently:

```python
# Lines ~78-79, 127-130, and others
except:
    continue  # Error swallowed silently
```

**Impact:** Failures in asset/timeframe selection will be invisible, making debugging nearly impossible.

**Recommended Fix:**  
@Coder: Replace bare `except:` with specific exception handling:
```python
except Exception as e:
    logger.warning(f"Non-critical error during scan: {e}")
    continue
```

### 2.3 🔴 CRITICAL – Missing Timeout in Socket Asset Selection

**Location:** `backend/services/gateway/main.py:147-172`  
**CORE_PRINCIPLES Violated:** #9 (Fail Fast)

**Finding:**  
The `select_asset` Socket.IO handler runs `subprocess.run()` without a timeout:

```python
# Line 161-165 - No timeout!
result = await asyncio.to_thread(run_script)
```

Compare with `/api/v1/select-timeframe` which correctly has `timeout=10`.

**Impact:** If asset_control.py hangs, the entire gateway event loop could be blocked.

**Recommended Fix:**  
@Backend-Specialist: Add timeout to the subprocess call:
```python
result = subprocess.run([...], capture_output=True, text=True, timeout=10)
```

## 3. HIGH Priority Issues

### 3.1 🟠 HIGH – Redundant Logic in Tick Aggregation

**Location:** `gui/Dashboard/src/hooks/useTickAggregation.js:86-88`

**Finding:**  
Redundant condition that can never be true:

```javascript
// Line 86-88
if (candle && typeof candle.time === 'number' && candleTime < candle.time && candleTime !== candle.time) {
  return;
}
```

If `candleTime < candle.time`, then `candleTime !== candle.time` is **always true**. This is dead logic.

**Impact:** Code confusion and potential maintenance issues.

**Recommended Fix:**  
@Coder: Simplify to:
```javascript
if (candle && typeof candle.time === 'number' && candleTime < candle.time) {
  return;
}
```

## 4. MEDIUM Priority Issues

### 4.1 🟡 MEDIUM – No Loading State for Timeframe Change

**Location:** `gui/Dashboard/src/store/marketStore.js:88-108`

**Finding:**  
`setSelectedTimeframe` makes a network request but has no loading indicator:

```javascript
setSelectedTimeframe: async (timeframe) => {

  // No loading state set
  try {
    const response = await fetch('http://localhost:8000/api/v1/select-timeframe', {...});
```

**Impact:** UI feels unresponsive during timeframe changes.

**Recommended Fix:**

### 4.2 🟡 MEDIUM – Timestamp Fallback May Mask Issues

**Location:** `gui/Dashboard/src/utils/validators.js:25`

**Finding:**  
Missing timestamp falls back to `Date.now()`:

```javascript
timestamp: Number.isFinite(timestamp) ? timestamp : Date.now()
```

**Impact:** Data without timestamps will appear valid but with incorrect time, potentially causing chart rendering issues.

**Recommended Fix:**  
@Coder: Add warning when fallback is used:
```javascript
if (!Number.isFinite(timestamp)) {
  console.warn('Market data missing timestamp, using current time');
}
```

---

### 4.3 🟡 MEDIUM – Inconsistent Error UX in TopBar

**Location:** `gui/Dashboard/src/components/TopBar.jsx:79-82`

**Finding:**  
Errors use `window.alert()` instead of the store's error pattern:

```javascript
} catch (err) {
  console.error('Ask AI failed:', err);
  window.alert(`Ask AI failed: ${err.message}`);
}
```

**Impact:** Inconsistent UX – other errors show in the banner, AI errors use alerts.

**Recommended Fix:**  
@Frontend-Specialist: Use the store's `lastError` pattern:
```javascript
set({ lastError: `Ask AI failed: ${err.message}` });
```

### 4.4 🟡 MEDIUM – High CPU in Collector Loop

**Location:** `backend/services/collector/main.py:54`

**Finding:**  
Very short sleep interval in main loop:

```python
time.sleep(0.1)  # 100ms = 10 iterations/second
```

**Impact:** High CPU usage when there's no data to process.

**Recommended Fix:**  
@Backend-Specialist: Consider adaptive sleep or longer interval:
```python
time.sleep(0.25)  # 250ms is usually sufficient for market data
```
