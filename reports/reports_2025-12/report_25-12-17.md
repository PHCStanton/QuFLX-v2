# Engineering Report: Legacy Data Collection & WebSocket Optimization
**Date:** 2025-12-17
**Author:** @Team-Leader
**Subject:** Analysis of Legacy Files and WebSocket Data Collection Strategy

## 1. Executive Summary
This report analyzes the critical dependency between Frontend UI interaction and WebSocket data streaming reliability in the Pocket Option environment. As highlighted in the prompt instructions, passive listening to the WebSocket is insufficient due to idle timeouts and "focus-only" streaming behaviors.

**Key Finding:** The system requires an "Active Driver" (UI Automation) to stimulate the backend, paired with a "Passive Collector" (WebSocket Listener) to capture the resulting data flood. The legacy files provide exactly this architecture.

## 2. Technical Analysis & Architecture (@Architect)

The solution relies on a Decoupled Producer-Consumer Pattern:

### Component A: The Stimulus (UI Automation)
**Files:** `favorites_select_topdown_collect.py` & `favorites_bar_scroll_select.py`
*   **Role:** Acts as the "Heartbeat" of the system.
*   **Mechanism:** Physically clicks assets and timeframe buttons in the DOM.
*   **Effect:**
    1.  Prevents WebSocket idle timeout (keep-alive).
    2.  Forces the backend to push immediate "Historical Data" payloads (~100 candles) for the selected context.
    3.  Ensures data stream is relevant to the active analysis target.

### Component B: The Collector (Data Ingestion)
**Files:** `data_streaming_csv_save.py`
*   **Role:** Passive listener and archiver.
*   **Mechanism:** Listens to `updateCharts` and other WebSocket events.
*   **Innovation:** `_detect_timeframe_from_candles` method.
    *   *Why it's critical:* When the UI clicks "H1", the WebSocket sends candles. The script calculates the time difference between candles (e.g., 3600s) to auto-classify them as "H1" data without needing explicit metadata tags. This makes it robust against protocol changes.

## 3. Detailed File Capabilities (@Engineer)

### 1. `data_streaming_csv_save.py` (The Collector)
*   **Status:** Production-Ready.
*   **Features:**
    *   Auto-detects timeframes based on data content (1m, 5m, 15m, 1h, 4h, 1d).
    *   Saves data to structured directories: `data/data_output/assets_data/data_collect/{TIMEFRAME}_candles/`.
    *   Handles the ~100 candle burst effectively.

### 2. `favorites_select_topdown_collect.py` (The Driver)
*   **Status:** Production-Ready.
*   **Features:**
    *   Iterates through the Favorites Bar.
    *   Performs "Topdown" selection: For each asset, it clicks H1 -> M15 -> M5 -> M1 (configurable).
    *   **Crucial Integration Point:** This script *must* run continuously to keep the data flowing to the Collector.

## 4. Operational Strategy & Workflow

To achieve the "Legacy Files Data Collecting Method," we must run a dual-process workflow:

1.  **Terminal 1 (The Ear):**
    ```powershell
    python capabilities/data_streaming_csv_save.py --stream
    ```
    *Action:* Waits for data.

2.  **Terminal 2 (The Hand):**
    ```powershell
    python capabilities/favorites_select_topdown_collect.py --all --td-delay-ms 2000
    ```
    *Action:* Clicks through assets and timeframes. The `td-delay-ms` is critical—it gives the WebSocket enough time to send the history and the Collector enough time to write the CSV before the UI switches context.

## 5. Next Step Recommendations

1.  **Integration Testing:** Verify the optimal `td-delay-ms`. If the UI switches too fast, the Collector might miss the tail end of the history payload.
2.  **AI Analysis Integration:** The prompt mentions combining this with AI image analysis. `favorites_select_topdown_collect.py` already supports `--save-screenshots`. We can pipe these images to an analysis agent.
3.  **Process Orchestration:** Consider a wrapper script (e.g., `run_collection_cycle.py`) that launches both processes and manages their lifecycle to ensure they stay in sync.

## 6. Conclusion
The legacy files provided are not just "old code"; they represent a specific, necessary workaround for the platform's constraints. We will proceed by treating `favorites_select_topdown_collect.py` as the primary driver for our data collection campaigns, ensuring it is always paired with the streaming listener.
