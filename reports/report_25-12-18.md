## 3. Delegated Reviews

### 3.1 Delegating to @Investigator: Forensic findings (read-only)
#### Summary
Multiple modules use `except Exception: pass` and bare `except:` in hot paths. This violates “Fail Loud” and can mask data loss, UI desync, and automation failures.

#### Critical Issues
- **CRITICAL:** Silent failure in data parsing (ticks/history) can drop market events without any trace.
- **HIGH:** Socket.IO “room” subscription exists but gateway emits aren’t routed to rooms.
- **HIGH:** Collector reads Chrome performance logs in multiple methods; log consumption + shared dedupe set can cause missed frames.

#### Detailed Findings (evidence)
- **Silent parse failures (ticks):** `backend/services/collector/interceptor.py:211-233`
  - Quote: `except Exception: pass` in `_parse_single_tick`.
  - Why it matters: failures to parse individual ticks become invisible, causing “missing candles/ticks” symptoms that are un-debuggable.

- **Silent parse failures (dict ticks):** `backend/services/collector/interceptor.py:235-254`
  - Quote: `except Exception: pass` in `_parse_single_tick_dict`.

- **Silent failures while scanning WebSocket frames (history):** `backend/services/collector/interceptor.py:22-70`
  - Quote: `except Exception: continue` (no log) inside the log loop.
  - Why it matters: history payload parsing errors are discarded, so the “~100 historical candles” burst can be lost silently.

- **Potential data loss via shared dedupe set:** `backend/services/collector/interceptor.py:20-50` and `:72-105`
  - Evidence: `self.processed_messages` is shared between `fetch_history_events()` and `fetch_ticks()`.
  - Risk: one method can mark an entry as processed, preventing the other from seeing it.

- **Over-logging in high-frequency path:** `backend/services/collector/interceptor.py:223-224`
  - Quote: `logger.info(f"Raw Tick: ...")`.
  - Why it matters: INFO logs per tick can degrade throughput and flood logs.

- **Silent Selenium failures:** `backend/services/gateway/asset_control.py:127-128`
  - Quote: `except Exception: pass` around the dropdown search input.
  - Why it matters: UI selectors change frequently; silent exceptions make root-cause analysis impossible.

- **Bare `except:` blocks (Selenium clicks):** `backend/services/gateway/asset_control.py:210-213`, `:244-248`
  - Quote: `except: driver.execute_script(...)`.
  - Why it matters: can hide genuine Selenium/WebDriver errors beyond “click intercepted”.

#### Recommendations
- Assign to **@Debugger + @Coder**: Replace silent `except` blocks with explicit errors/logging and structured error propagation.
- Assign to **@Optimizer**: Reduce per-tick logging; consider sampling or debug-level logs.
- Assign to **@Engineer**: Rework interceptor log consumption/deduplication so history and ticks are captured deterministically.

#### Risk Forecast
If left unresolved, next failures will manifest as random “missing candles”, “no history captured”, and “asset/timeframe selection sometimes works” issues that cannot be reproduced reliably.

### 3.3 Delegating to @Architect: Architecture & flow risks
#### Findings
- **HIGH:** Socket.IO “room” abstraction is currently unused/misapplied.
  - Evidence: `backend/services/gateway/main.py:131-136` enters room `market_data:{asset}`.
  - Evidence: `backend/services/gateway/main.py:88-99` emits using event names (`market_data:{asset}`) instead of emitting `market_data` to a room.
  - Effect: frontend room subscriptions don’t gate traffic; asset isolation is happening client-side.

- **MEDIUM:** Gateway is a monolith of Redis listener + API + Selenium subprocess orchestration.
  - Risk: difficult to test and reason about failure domains.

#### Recommendations
- Assign to **@Engineer + @Backend_Specialist**: Normalize the contract:
  - Either (A) keep a single `market_data` event and emit to rooms, or (B) keep global broadcast and remove room complexity.

### 3.4 Delegating to @Engineer: Functional issues & simplification opportunities
#### Findings
- **HIGH:** Timeframe correctness is not enforced end-to-end.
  - Evidence: `gui/Dashboard/src/components/ChartWorkspace.jsx:118-121` hardcodes `interval = 60` with a TODO to map `selectedTimeframe`.
  - Risk: UI can show non-1m timeframe label while rendering 1m aggregation.

- **MEDIUM:** Collector parsing has multiple formats, but error paths are not explicit.
  - Evidence: `backend/services/collector/interceptor.py:130-151` uses bare `except:` in base64 decode fallback.

#### Recommendations
- Assign to **@Frontend-Specialist**: Keep timeframe options pinned to `1m` in the Combobox until mapping is complete.
- Assign to **@Coder**: Enforce a single internal “chart timeframe seconds” value derived from `selectedTimeframe`.
