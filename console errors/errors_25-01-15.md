## 🔍 Investigation Complete: Console Errors Analysis

I've thoroughly investigated both console errors. Here are my findings:

### **Error 1: "Object is disposed" from lightweight-charts**

**Root Cause:** Race condition between chart disposal and ResizeObserver callbacks.

**When it happens:**
1. Sidebar toggle triggers CSS width transition (`w-64` ↔ `w-16`)
2. ResizeObserver fires during the transition
3. If an oscillator was loaded but then removed (or is being unmounted), the chart's internal `DevicePixelContentBoxBinding` has been disposed
4. The resize callback still tries to call `chart.applyOptions({ width, height })` on the disposed chart

**Problem locations in OscillatorChart.jsx:**
```javascript
// Line 82-87: ResizeObserver continues firing even after chart.remove()
const resizeObserver = new ResizeObserver((entries) => {
  // NO CHECK: chartRef.current could be null or disposed
  const { width, height } = entries[0].contentRect;
  chart.applyOptions({ width, height }); // CRASH if disposed
});
```

**The cleanup at line 89-97 sets `chartRef.current = null`, but:**
- ResizeObserver callbacks can fire AFTER cleanup starts
- There's no "disposed" flag to check before applying options

---

### **Error 2: "Failed to sync oscillator time scale Error: Value is null"**

**Root Cause:** `mainChart.timeScale().getVisibleRange()` returns `null` when chart has no data.

**Problem location in OscillatorChart.jsx (lines 113-137):**
```javascript
const sync = (range) => {
  // Line 114-117: This guard catches null range, but...
  if (!range || typeof range.from !== 'number' || typeof range.to !== 'number') {
    return;
  }
  // ...the error "Value is null" comes from oscTimeScale.setVisibleRange() 
  // being called with what appears valid but isn't
  oscTimeScale.setVisibleRange(range); // Line 123 - throws "Value is null"
};

// Line 137: Initial sync triggers with potentially null range
setTimeout(() => {
  sync(mainTimeScale.getVisibleRange()); // getVisibleRange() returns null!
}, 100);
```

**The issue:** `getVisibleRange()` can return null even after 100ms timeout if:
- Main chart has no data yet
- Chart is in a transitional state during unmount
- Time scale hasn't been painted yet

## 📋 Recommended Fixes

### **Fix 1: Add disposal guard to OscillatorChart.jsx**

```javascript
// Add a disposed ref
const isDisposedRef = useRef(false);

// In ResizeObserver callback:
const resizeObserver = new ResizeObserver((entries) => {
  if (isDisposedRef.current || !chartRef.current) return; // Guard
  if (entries.length === 0 || !entries[0].contentRect) return;
  const { width, height } = entries[0].contentRect;
  try {
    chartRef.current.applyOptions({ width, height });
  } catch (err) {
    // Chart may be disposed - ignore resize errors
    if (!err.message?.includes('disposed')) {
      console.error('Resize error:', err);
    }
  }
});

// In cleanup:
return () => {
  isDisposedRef.current = true; // Set BEFORE disconnect
  resizeObserver.disconnect();
  if (chartRef.current) {
    chartRef.current.remove();
  }
  // ...rest of cleanup
};
```

### **Fix 2: Add robust null check in sync function**

```javascript
const sync = (range) => {
  // Enhanced guard - check chart is still valid
  if (isDisposedRef.current || !chartRef.current) return;
  
  // Null check for range
  if (!range) return;
  if (typeof range.from !== 'number' || typeof range.to !== 'number') return;
  if (!Number.isFinite(range.from) || !Number.isFinite(range.to)) return;
  if (range.from >= range.to) return;
  
  try {
    const oscTimeScale = chartRef.current.timeScale();
    if (!oscTimeScale) return;
    oscTimeScale.setVisibleRange(range);
  } catch (err) {
    // Only log non-disposed errors
    if (!err.message?.includes('disposed') && !err.message?.includes('null')) {
      console.error('Failed to sync oscillator time scale', err);
      if (onError) {
        onError(`Oscillator chart sync failed: ${err.message}`);
      }
    }
  }
};
```

### **Fix 3: Delay initial sync until main chart has data**

```javascript
// Replace setTimeout with a proper data-ready check
const timeoutId = setTimeout(() => {
  if (isDisposedRef.current || !mainTimeScale) return;
  const range = mainTimeScale.getVisibleRange();
  if (range) { // Only sync if range exists
    sync(range);
  }
}, 100);
```
