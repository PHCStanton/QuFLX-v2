# 1. CONSOLE ERRORS:

The key part of the error is:

Code
Uncaught Error: Cannot update oldest data, last time=[object Object], new time=[object Object]
This comes from lightweight-charts, specifically inside its DataLayer._internal_updateSeriesData method.

Lightweight Charts expects each data point in a series to have a time field that is strictly increasing (newer data must have a later timestamp than the previous one).

The error is thrown when you try to update the oldest data point with a new time that is not valid (e.g., equal or earlier than the existing one).

The [object Object] part indicates that instead of a primitive (like a number or string timestamp), you’re passing a raw JavaScript object as the time field. The library doesn’t know how to compare two objects, so it fails.

⚠️ Likely causes
Incorrect time format

You may be passing a Date object directly (new Date()) instead of a number (Date.getTime()) or a string (YYYY-MM-DD).

Lightweight Charts requires time to be either:

A business day object ({ year: 2025, month: 12, day: 2 }), or

A UNIX timestamp (number of seconds since epoch).

Updating the wrong data point

The error mentions "oldest data" — meaning you’re trying to overwrite the very first point in the series with a new time. That’s not allowed; you should append new data or update existing points with the same time.

React hook effect timing

The stack trace shows ChartWorkspace.jsx:140:31 inside a React effect (commitHookEffectListMount).

This suggests you’re calling series.update() inside a useEffect or similar hook, but the data being passed has malformed or inconsistent time.

✅ How to fix it
Here’s how you can resolve the issue:

Ensure correct time format

js
// ❌ Wrong: passing a Date object
series.update({ time: new Date(), value: 100 });

// ✅ Correct: pass a timestamp (seconds)
series.update({ time: Math.floor(Date.now() / 1000), value: 100 });

// ✅ Or use business day format
series.update({ time: { year: 2025, month: 12, day: 2 }, value: 100 });
Do not update the oldest data point with a new time

If you want to change the value of an existing point, keep the same time.

If you want to add new data, make sure the time is strictly greater than the last one.

js
// Update existing point
series.update({ time: 1733145600, value: 105 }); // same time, new value

// Append new point
series.update({ time: 1733232000, value: 110 }); // later time
Validate your data before updating

js
function safeUpdate(series, point) {
  if (typeof point.time === "object" && !(point.time.year && point.time.month && point.time.day)) {
    throw new Error("Invalid time format for lightweight-charts");
  }
  series.update(point);
}
